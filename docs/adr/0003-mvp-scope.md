# ADR 0003: MVP スコープ定義

- **日付**: 2026-04-16
- **状態**: Accepted

## 背景

「MVP」の意味が、プロダクト文脈で「最小限で配布可能」と「現場で使い始められる
必須機能セット」で揺れていたため、ここで明示的に定義する。

ver.2 の MVP は **「配布先の医療機関で v1 を置き換えて日常運用を開始できる機能
セット」** とする。最小動作ではなく、**現場投入に耐える最小セット**。

## 決定: MVP 機能セット (v1.0)

### 含める

#### 端末 (打刻)
- NFC カードスキャン (PaSoRi)
- 打刻種別の自動推定 (`DefaultPunchPolicy`)
- 確認 UI (氏名・時刻・推定種別表示、直近履歴表示)
- 30 秒カウントダウン自動確定、OK 1 秒長押しでスキップ
- 種別変更 / キャンセル
- 同一カード連続スキャン無視 (既定 5 秒)
- 未登録カード検出時のログ記録 (管理者モードでのみ紐付け可)
- NTP 同期チェック (±10 秒超で打刻無効)
- オフライン時の local cache + 復旧時再送

#### Server (コア機能)
- 従業員管理 (追加 / 編集 / 無効化)
- カード紐付け管理
- 打刻保存 (UUID v7 punch_id の UNIQUE 制約で冪等性保証)
- 月次勤怠表閲覧 (日別ペアリング、勤務時間、不整合フラグ)
- 打刻修正 (理由必須、監査ログ記録)
- シフト管理 (作成 / 公開 / 編集、Excel 非依存)
- シフトと実績の差分検出 (打刻漏れ、シフト外打刻)
- 監査ログ (広範囲、append-only)
- 日次バックアップ (30 日保持)
- v1 データ import (別 CLI、フル履歴)

#### Admin Web
- 管理者ログイン (Session + Cookie、24h + 活動延長)
- 上記 Server 機能の UI

#### LINE WORKS 連携 (**フル双方向**)
- **送信**: 打刻漏れ通知 / 異常通知 / シフト公開通知 / 給料概算配信
- **受信**: 照会 (自分の勤怠・シフト) + 修正申請
- 承認フロー (`requested` → `auto_approved` or `approved` / `rejected` → `applied`)
- Bot callback の HMAC 署名検証必須

### 含めない (post-MVP)

- 給与計算エンジン
- 複雑な休憩体系 (BreakStart/End など、trait だけは用意)
- 丸め規則の UI 設定 (`RoundingPolicy` の trait は用意、実装は NoRounding のみ)
- 複数拠点同期
- モバイルネイティブアプリ
- 顔認証 / 生体認証
- コード署名 (v1.0 リリース直前に判断)
- Sentry / GlitchTip によるエラートラッキング (MVP はローカルログのみ)
- 高度なシフト自動作成エンジン

## 受け入れ基準 (MVP v1.0)

以下をすべて満たせば v1.0 リリース候補。

1. macOS 開発環境で実機 PaSoRi により打刻できる
2. Windows / macOS / Ubuntu 向けに Tauri ビルドが通る
3. Raspberry Pi で Server バイナリが systemd で常時稼働する
4. 1 枚の登録済みカードで、出勤 → 退勤 → 翌日出勤が正しく動く
5. 未登録カードが誤って一般打刻されない
6. 管理者が Web UI から従業員追加、カード紐付け、打刻修正を行える
7. 月次勤怠表が Admin Web で確認できる
8. 打刻修正に理由と監査ログが残る
9. LINE WORKS 送信が動作する (4 種類の通知)
10. LINE WORKS 受信で照会と修正申請が動作する
11. Cloudflare Tunnel 経由で LINE WORKS callback を受信できる
12. 通知失敗時も打刻自体は成功する
13. Server 停止時、Terminal は local cache に打刻を溜め、復旧時に自動再送する
14. NTP 同期がずれた Terminal は打刻を拒否する
15. v1 SQLite から全履歴を import_v1 CLI で取り込める
16. 日次バックアップが自動実行され、30 日分保持される
