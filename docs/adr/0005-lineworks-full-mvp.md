# ADR 0005: LINE WORKS 双方向連携を MVP に含める

- **日付**: 2026-04-16
- **状態**: Accepted
- **関連**: ADR 0002 (C プラン), ADR 0003 (MVP スコープ)

## 背景

当初は「MVP は送信のみ、受信 (照会・申請) は v1.1 以降」の段階リリースを
検討していた (他 LLM からの助言は A+ プラン推奨)。しかし以下の理由で MVP に
フル双方向を含めることに決定した。

1. 修正申請は v1 運用で実際に困っている機能
2. 管理者が毎回対面で修正を受けるのは現場負担
3. 「v1.1 に回した機能は実際には回らない」リスク
4. サーバーを立てた以上、受信追加の追加コストは大きくない
5. LINE WORKS は既存の職場アカウントで完結するので、新規サービス登録のハードルがない

## 決定

MVP (v1.0) に以下を含める。

### 送信 (Server → LINE WORKS)
- 打刻漏れ疑い通知
- 異常通知 (未登録カード検出、NTP ずれ、リーダー切断等)
- シフト公開通知
- 給料概算の定期配信

### 受信 (LINE WORKS → Server callback)
- 照会: 自分の今日 / 今月の勤怠、今日 / 今月のシフト
- 修正申請: 打刻漏れ、軽微修正
- 承認フロー: `requested` → `auto_approved` / `approved` / `rejected` → `applied`

## 実装方針

- Server は Cloudflare Tunnel で外部公開
- callback URL に LINE WORKS からの POST を受ける
- `X-WORKS-Signature` の HMAC 署名検証を必須化 (未検証は 401)
- Bot token / client secret は Bitwarden CLI で起動時注入
- コマンドは UX 順に (1) 固定メニュー → (2) postback ボタン → (3) 定型コマンド → (4) 自由入力
  の優先順位。自由入力は最後の手段

### 権限

**本人が自動反映できる操作**:
- 自分の勤怠 / シフト照会
- 当日分の打刻漏れ申請
- 当日中の軽微修正申請

**管理者承認が必要**:
- 過去日修正
- 締め済み期間修正
- 他人の勤怠変更
- シフト確定 / 公開
- カード再紐付け

## 失うもの / 得るもの

### 得るもの
- v1 で困っていた修正申請が MVP から使える
- 遠隔照会が MVP で提供できる
- 「複数端末アクセス」の副産物 (Admin Web が複数端末から使える)

### 失うもの
- MVP 実装工数が増える (特に承認フロー、Bot 対話 UX)
- 運用責任が増える (Cloudflare Tunnel、LINE WORKS Bot 設定、Bitwarden アカウント)

これは ADR 0002 の C プラン採用でカバーされる範囲と判断。

## 代替案と却下理由

- **送信のみで MVP**: v1 の運用課題が解決しない。
- **Slack Socket Mode / Telegram**: 技術的には gateway 不要で楽。ただし LINE WORKS は既存職場ツールなので業務要件と整合する。乗り換えメリットが薄い。

## 禁止事項

- LINE WORKS メッセージだけで無制限な管理者権限を与えない
- 受信自然文の曖昧な解釈で DB を直接書き換えない
- 承認なしに締め済みデータを書き換えない
- LINE WORKS 連携失敗を打刻失敗に連鎖させない
