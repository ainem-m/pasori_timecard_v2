# ADR 0009: テスト識別子と言語ポリシー

- **日付**: 2026-04-16
- **状態**: Accepted
- **関連**: AGENTS.md §6, ADR 0004

## 背景

これまで本プロジェクトでは「テスト名は日本語で仕様を表現する」という運用を
採用していた。しかし Rust の `#[test] fn ...` のように**識別子そのもの**が
テスト名になる系統では、日本語識別子が以下の実務上の摩擦を生む。

- ツール連携やフィルタ実行で ASCII 前提の操作がしづらい
- grep / ripgrep / shell 補完で扱いが重くなる
- エディタや周辺ツールによっては表示やコピーで不安定になる

一方で、仕様を日本語で読める状態は維持したい。

## 決定

### Rust など識別子ベースのテスト

- テスト関数名は **ASCII の英語 `snake_case`** とする
- 直前に **日本語コメント** を置き、そのコメントで仕様を表現する
- 必要なら Arrange-Act-Assert / Given-When-Then の区切りコメントも日本語で書く

例:

```rust
#[test]
// 最終打刻が前日なら次は出勤と推定する。
fn decides_clock_in_when_last_event_was_on_previous_day() { ... }
```

### Vitest / Jest / Playwright など表示名ベースのテスト

- `it(...)`, `test(...)`, `describe(...)` の**表示文字列は日本語**で仕様を表現する
- コールバック関数の変数名や補助関数名は英語でよい

例:

```typescript
it('締め日が15日なら前月16日から当月15日までが当月扱いになる', () => { ... })
```

## 結果

- 仕様の可読性は日本語で維持する
- 識別子ベースのテストはツールフレンドリーにする
- Definition of Done は「テスト名が日本語」ではなく
  「**テストの仕様記述が日本語で読める**」に読み替える

## 代替案と却下理由

- **現行どおり全テストを日本語識別子にする**:
  仕様可読性は高いが、識別子ベース言語との相性が悪い
- **すべて英語に統一する**:
  ツール面は有利だが、業務仕様の読みやすさを落とす
