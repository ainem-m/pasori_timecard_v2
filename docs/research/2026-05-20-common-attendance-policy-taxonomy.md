## 観察した共通パターン

KING OF TIME、freee 勤怠管理 Plus、マネーフォワード クラウド勤怠、SmartHR 勤怠管理の公開ヘルプを見ると、各社の表現は違うが、共通して次の軸で勤怠計算を分解している。

| 軸 | よくある項目 | 備考 |
|---|---|---|
| 対象者グループ | 雇用区分、就業ルール、所属、個人別設定 | 「正社員」「パート」「ドクター」のような区分はこの層。 |
| 勤務予定 | スケジュール、シフト、勤務パターン、所定開始/終了、休憩予定 | 遅刻・早退・所定外・残業の判定基準になる。 |
| 打刻種別 | 出勤、退勤、休憩開始、休憩終了、外出/戻り | MVP の `ClockIn/ClockOut` だけだと休憩・外出 policy は表現しにくい。 |
| 丸め | 出勤/退勤/休憩開始/休憩終了ごとの丸め、日別集計値の丸め | 打刻原本は保持し、計算/表示だけに適用する設計が重要。 |
| 休憩 | 予定休憩、打刻休憩、労働時間に応じた自動休憩、休憩不足アラート | 自動控除と法定休憩警告は別 policy。 |
| 残業 | 固定基準時間、予定超過、所定外、法定外、深夜、休日、申請済みのみ | 「残業」の意味が職場ごとに違う。名前付けが重要。 |
| 休日 | 法定休日、所定休日、休日出勤、振休、代休 | 法定休日と所定休日の扱いを混ぜない。 |
| 休暇 | 有給、半休、時間休、独自休暇、みなし勤務時間 | 有給付与/残数管理と、月次表の取得日数表示は分ける。 |
| 申請承認 | 残業申請、休日出勤申請、打刻修正、休暇申請、シフト変更 | 未申請の早出/残業を計算対象にするかが policy になる。 |
| 異常検知 | 打刻漏れ、重複休憩、休憩不足、長時間労働、36 協定アラート | 最初は警告として持つ方が安全。 |
| 出力 | 給与連携 CSV、月次帳票、部門別集計、個人確認画面 | 正本 DB と帳票互換 policy は分離する。 |

## 法令由来で固定に近いもの

設定で無効化できる「便利機能」にしてはいけないもの。ただし、システムが給与計算まで行わないなら、まずは警告・分類・監査の根拠として扱う。

| 項目 | 内容 | 実装上の扱い |
|---|---|---|
| 客観的記録 | 始業・終業時刻は労働日ごとに確認・記録する。IC カード等の客観的記録は原則的方法の一つ。 | `punch_event` は raw を保存し、丸め後時刻で上書きしない。 |
| 法定労働時間 | 原則 1 日 8 時間、1 週 40 時間。 | 法定外時間の分類・警告候補。給与計算は MVP 外。 |
| 法定休憩 | 6 時間超 45 分以上、8 時間超 1 時間以上。 | 自動控除ではなく、休憩不足疑いの warning policy。 |
| 休日 | 毎週 1 日または 4 週 4 日以上。 | 休日設定/シフトと連携した warning policy。 |
| 割増分類 | 時間外、深夜、休日、月 60 時間超など。 | export 用分類候補。割増賃金計算は MVP 外。 |
| 年次有給 | 条件を満たす労働者に付与。パートも比例付与対象。 | 残数管理は後続。MVP は取得日数表示まで。 |

## よくある policy preset 候補

ここでいう preset は「複数の小 policy とパラメータを束ねた名前付き設定」。
設定画面でゼロから条件ビルダーを組ませるのではなく、まず preset を選ばせ、必要な値だけ調整する。

### 1. `actual_time_basic`

実打刻ベースの最小構成。

| 項目 | 値 |
|---|---|
| 対象 | 小規模事業所、勤怠記録のみ、給与計算は外部 |
| 勤務時間 | 出勤から退勤までの実時間を合算 |
| 丸め | なし |
| 休憩 | 打刻休憩があれば控除。なければ控除なし |
| 残業 | 8h 超/週 40h 超を参考分類 |
| 強み | 法令上の客観的記録と相性がよい |
| 弱み | 既存 Excel の 30 分切り上げ運用とは異なる |

### 2. `hourly_with_rounding`

パート・アルバイト向けの時給集計寄り。

| 項目 | 値 |
|---|---|
| 対象 | パート、アルバイト、時給スタッフ |
| 勤務時間 | ペアごとの差分合算 |
| 丸め | 出勤切り上げ、退勤切り捨て/丸めなし、単位は 5/10/15/30 分など |
| 休憩 | 予定休憩または労働時間に応じた自動休憩 |
| 残業 | 8h 以内/8h 超、または契約時間超 |
| 強み | 多くの職場要望を吸収しやすい |
| 注意 | raw punch を丸めて保存しない。集計用時刻を別に持つ |

現在の院内 `パート` テンプレートはこの派生で、出勤だけ 30 分切り上げ、退勤丸めなし、8h 内/8h 超に分ける。

### 3. `scheduled_regular`

所定時刻・スケジュールを基準にする正社員向け。

| 項目 | 値 |
|---|---|
| 対象 | 固定勤務の正社員 |
| 勤務時間 | 予定時刻と実打刻を比較 |
| 遅刻/早退 | 出勤予定・退勤予定との差分 |
| 残業 | 退勤予定後、または固定基準時刻後 |
| 休憩 | 予定休憩を自動適用、または打刻休憩 |
| 申請 | 未申請残業を対象外/警告/対象内にする選択 |

現在の院内 `正社員` テンプレートはこの派生だが、残業基準は法定外ではなく `19:00` 以降表示。

### 4. `shift_based`

日ごとに勤務パターンが変わる職場向け。

| 項目 | 値 |
|---|---|
| 対象 | 医療、介護、店舗、シフト勤務 |
| 勤務時間 | 日別シフトの予定開始/終了を基準 |
| 遅刻/早退 | 当日のシフトとの差分 |
| 残業 | シフト予定超過、法定外、休日を分ける |
| 休憩 | シフトに休憩予定を含める |
| 注意 | shift_assignment との統合が必要 |

### 5. `attendance_days_only`

勤務日数・出勤事実中心で、時間集計を主目的にしない職種向け。

| 項目 | 値 |
|---|---|
| 対象 | ドクター、業務委託的運用、時間給与でない職種 |
| 勤務時間 | 表示しない、または参考値のみ |
| 集計 | 出勤日数、有給日数、要確認 |
| 残業 | なし、または参考警告のみ |
| 注意 | 労働者性や法令適用除外をシステムが断定しない |

現在の院内 `ドクター` テンプレートはこの系統。

### 6. `flex_time`

フレックスタイム制向け。MVP では後回し。

| 項目 | 値 |
|---|---|
| 対象 | コアタイム/フレキシブルタイムがある職場 |
| 勤務時間 | 清算期間で過不足を計算 |
| 遅刻/早退 | コアタイム違反など別軸 |
| 残業 | 清算期間の総労働時間超過 |
| 注意 | 月次集計が複雑。MVP に入れない方がよい |

### 7. `variable_working_hours`

1 か月単位/1 年単位の変形労働時間制向け。MVP では後回し。

| 項目 | 値 |
|---|---|
| 対象 | 繁閑差がある職場 |
| 勤務時間 | 期間平均で法定枠を判断 |
| 必須データ | 事前に定めた勤務カレンダー |
| 注意 | 法令・就業規則との整合が強く必要 |

### 8. `overnight_shift`

日跨ぎ勤務向け。医療・介護・警備・飲食で出やすい。

| 項目 | 値 |
|---|---|
| 対象 | 夜勤、宿直、深夜勤務 |
| 日付帰属 | 出勤日基準/退勤日基準/締め時刻基準 |
| 深夜 | 22:00-05:00 を別分類 |
| 休憩 | 夜勤休憩を打刻または予定で控除 |
| 注意 | 現行 v2 の同日ペアリングだけでは不足 |

## 小 policy への分解案

preset は以下の小 policy を組み合わせる形にすると、設定 UI と Rust 実装の両方が保ちやすい。

| policy | 役割 | 設定画面で扱うか |
|---|---|---|
| `PeriodPolicy` | 締め期間を決める | yes |
| `DayBoundaryPolicy` | 日跨ぎ勤務の帰属日を決める | later |
| `PairingPolicy` | 打刻列から勤務区間/不整合を作る | mostly fixed |
| `RoundingPolicy` | 集計用時刻・日別集計値を丸める | yes |
| `BreakPolicy` | 打刻休憩/予定休憩/自動休憩を処理する | yes |
| `SchedulePolicy` | 予定時刻・所定時間を与える | yes |
| `OvertimePolicy` | 所定外/法定外/固定時刻後/申請済みのみを分類する | preset + params |
| `HolidayPolicy` | 法定休日/所定休日/通常日を分類する | yes |
| `LeavePolicy` | 有給/半休/時間休/みなし勤務を扱う | preset + params |
| `WarningPolicy` | 打刻漏れ、休憩不足、長時間、重複などを検出する | yes |
| `ExportPolicy` | 給与 CSV/帳票列へ写像する | preset + plugin candidate |

## 設定 UI で扱うべきもの

UI で安全に扱えるのは、数値・選択肢・ON/OFF として説明できるもの。

- 締め日: 15 日/月末など。
- 雇用区分ごとの preset 選択。
- 出勤/退勤/休憩開始/休憩終了の丸め単位と方向。
- 残業基準: 8h 超、所定超、固定時刻後、申請済みのみ。
- 固定時刻: 例 `19:00`。
- 休憩: 打刻休憩、予定休憩、自動休憩。
- 自動休憩の閾値: 6h 超 45m、8h 超 60m など。
- 半休換算: 0.5 日。
- 打刻漏れや休憩不足の warning 有効/無効。

## UI だけで扱わない方がよいもの

条件ビルダー化すると複雑になり、監査性・説明可能性が落ちるもの。

- 複数の残業分類をまたぐ複雑な優先順位。
- 夜勤の日付帰属。
- 変形労働時間制やフレックス清算。
- 給与ソフトごとの CSV 列変換。
- 職場独自の帳票列。
- 「この職種だけ、この曜日だけ、この備考のときだけ」の連鎖条件。

これらは preset 追加、Rust policy 追加、または将来の読み取り専用 plugin に寄せる。

## 独自スクリプト/プラグインを入れるなら

いきなり任意コード実行を入れるのは避ける。
勤怠システムでは、正本打刻・修正履歴・監査ログの整合性が壊れるリスクが高い。

安全な拡張境界:

| 拡張点 | 許可 | 禁止 |
|---|---|---|
| 集計後 processor | 月次集計 DTO を読み、追加列・警告・export 値を返す | DB 書き換え、punch_event 変更 |
| export plugin | 既存集計を給与 CSV/Excel に変換する | 集計ロジックの隠れ変更 |
| warning plugin | 追加の注意喚起を返す | 打刻を却下する、修正を自動適用する |
| report template | 帳票レイアウトを変える | 正本データの変更 |

plugin 入力は `MonthlyTimesheet` と `Employee`、出力は `DerivedColumn[]` / `Warning[]` / `ExportFile` 程度に制限する。
打刻修正は必ず core use case 経由にし、audit を通す。

## MVP での推奨

MVP ではスクリプト基盤ではなく、以下を先に固める。

1. `actual_time_basic`
2. `hourly_with_rounding`
3. `scheduled_regular`
4. `attendance_days_only`

この 4 つで、現在の院内 `パート` / `正社員` / `ドクター` は表現できる。
`shift_based` は shift 機能の完成後に追加する。
`flex_time`、`variable_working_hours`、`overnight_shift` は ADR を分ける。

## 設計上の重要な判断

- raw 打刻は不変の正本として保存する。
- policy は raw から derived timesheet を作る。
- 丸め済み時刻を `punch_event.occurred_at` に保存しない。
- 法令由来 warning と院内帳票 policy を分ける。
- `残業` という単語だけでは曖昧なので、内部では `scheduled_overtime`、`statutory_overtime`、`fixed_time_overtime` のように分ける。
- UI は preset 選択 + 主要パラメータ編集に留める。
- plugin は最初から入れず、将来のために `ExportPolicy` / `DerivedColumnPolicy` の境界だけ意識する。
